3 test faze jednoho testu vs 3 samostatne testy?

1) input validation
call: factorial aaa

    - program by nemel skoncit s tracebackem
    - program by mel skoncit s peknou error hlaskou a exit codem 10

2) spatny test
call: factorial <int>

    - test bude spatne napsany, bude ocekavat jinou hodnotu nez factorial bude vracet,
      ale chyba bude v testu a ne v programu
    - bude na userovi aby test opravil. test by mel byt dobre komentovany at je userovi
      jasne kde ma co zmenit

3) corner cases
call: factorial 0
      factorial -1

    - factorial se bude chovat spatne, je na userovi aby factorial opravil
    - ocekavana hodnota pro factorial 0 je 1
    - ocekavana hodnota pro factorial -1 je <not_defined>, exit code 11

v kazde fazi/testu by meli jeste nejake jine asserty at se toho testuje vic
a vypada to lepe, ale zase ne moc, at se v tom user neztracui
